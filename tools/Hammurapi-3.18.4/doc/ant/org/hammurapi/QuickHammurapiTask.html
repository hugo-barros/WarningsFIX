hammurapi

Description

Performs automatic code reviews. Quick mode - doesn't store anything to the database, package and repository level inspectors are not invoked.

Example

If you copy content of Hammurapi lib directory to ant lib directory then you can invoke Hammurapi in the following way: 
 <taskdef name="quickurappi" classname="org.hammurapi.QuickHammurapiTask" />
 

 <quickurappi>
     <src dir="src"/>
     <output dir="review"/>
 </quickurappi>
or, if you didn't copy jar files to Ant lib directory, use this syntax: 
 <taskdef name="quickurappi" classname="org.hammurapi.QuickHammurapiTask">
     <classpath>
         <fileset dir="${hammurapi.home}/lib" includes="*.jar"/>
     </classpath>
 </taskdef>
 

 <quickurappi>
     <src dir="src"/>
     <output dir="review"/>
 </quickurappi>

Implementing Class

org.hammurapi.QuickHammurapiTask

Attributes

Attribute Description Required
unpackDir If this attribute is set then HAR archive will be unpacked in the given directory instead of a temporary one. No.
debugType Class name to debug No.
embeddedInspectors Load embedded inspectors. Defaults to true. No.
classPath No.
archive Hammurapi archive to process. No.
dpmoThreshold Fail build if project DPMO is above the threshold. No.
sigmaThreshold Fail build if project Sigma is below the threshold. No.
failOnFirstException Fail build as soon as there is an exception. Default is false. No.
failOnWarnings Fail build if there have been warnings. Default is true. No.
severityThreshold Fail build on violations with severity levels lower or equal to the threshold. No.
title No.
waiverStubs File to output waiver stubs for rejected waiver requests to. Selected waiver stubs can then be copied to waiver source. Simplifies waiver creation No.
evictBadInspectors Remove inspector from inspector set if it throws an exception during review No.
force Force review even if the file is not changed No.
baselining Sets baselining mode. Possible values: off (default) - no baselining, on - do not report violations stored in the baseline table, set - all violations from current scan are saved to the baseline table. The idea is to filter out all violations in preexisting code and report only new violations. Not all violations can be filtered out, only thouse with signatures. Significant code modifications can surface some baselined violation. No.
forceOnWarnings Force review of files with warnings, even if the file is not changed. Default is true No.
reviewDescription Description of review, e.g. release number. Appears in history annotation. No.
database If this parameter is set then Hypersonic standalone database will be used instead of temporary database. You must set database name if you want to run incremental reviews. No.
tabSize Tab size in source files. Defaults to 8. No.
