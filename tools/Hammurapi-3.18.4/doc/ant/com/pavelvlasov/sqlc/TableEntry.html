Implementing Class

Attributes

Attribute Description Required
catalog Catalog. Any catalog matches if not set No.
schema Schema. Any schema matches if not set No.
table Table. Any table matches if not set No.
qualifiedName Indicate where to qualify table name with schema name. Valid values are - 'none', 'table', 'method', and 'both' No.
generateMutators If 'true' (default) then mutators (setters) are generated along with accessors (getters) in row interfaces. No.
paramPerColumnInsert If 'true' (default) insert() method is generated for table with all columns listed as method parameters. For tables with big number of columns it is better off to set this attribute to false and use insert(<RowInterface>) method. No.
useSqlTypes If true then generated classes will use setObject(int, Object, int) method instead of setObject(int, Object) to set parameters of object type. Inherits value from task. Default value is true. No.
smartBase Base class (fully qualified name) for generated smart implementation. Default is com.pavelvlasov.sql.DatabaseObject The base class is supposed to have same constructors and methods as DatabaseObject. The idea is to subclass DatabaseObject to introduce additional qualities like audit, metrics, security, ... whithout changing class' contract. If this value is not set explicitly it gets inherited from the task. No.
