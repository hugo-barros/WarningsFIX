Attributes

Attribute Description Required
hasNullableParameters Set it to true if query contains primitive type parameters which can be 'NULL' and engine will be generated with primitive type wrapper classes as parameters. E.g. java.lang.Integer instead of int. No.
parameters Sets parameter names and types. Useful with not fully compilian JDBC drivers. Comma separated list of parameters in Java format, e.g. int a, long b, java.lang.String c. Type is optional. Class names must be fully qualified. Specifications can be skipped, e.g. int a,,java.lang.String c Type can include SQL type as defined in java.sql.Types. E.g. int:INTEGER, both sides are optional. Nullability can be specified using 'nullable' and 'not nullable' modifiers. Full types grammar looks like: 
 parameters = parameter? (, parameter)* parameter = name | (typeDefinition name ("nullable" | "not nullable")?) typeDefinition = javaType | (javaType? ":" sqlType?)
An alternative way to override parameter types is to use nested parameter element. Nested elements take precedence over parameters attribute. 		No.

description Statement description No.
useSqlTypes If true then generated classes will use setObject(int, Object, int) method instead of setObject(int, Object) to set parameters of object type. Inherits value from task. Default value is false. No.
